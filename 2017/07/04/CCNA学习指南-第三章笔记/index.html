CCNA学习指南-第三章笔记

July 04 2017

第三章 子网划分、变长子网掩码（VLSM）和TCP/IP措施

1.子网划分的好处

缩减网络流量、优化网络性能、简化管理和可以更灵活地形成大覆盖范围的网络

2.什么是子网掩码（subnet mask）

子网掩码是一个32位的值，通过它，接收IP数据包的一方可以从IP地址的主机号部分中区分出子网ID号地址

3.A类，B类，C类网络的默认子网掩码

对于A类网络，子网掩码最小值必须是255.0.0.0；B类网络，必须起始于255.255.0.0；C类网络，必须起始于255.255.255.0

4.无类的内部域路由（CIDR）的概念

CIDR是因特网服务提供商（ISP）为公司、家庭分配大量地址的基本方法，他们在某个成块的区域中提供地址

5.最大可用子网掩码数

最大的可用（不考虑地址的类）子网掩码只能是/30，因为你必须为主机位保留至少2位。斜线符（/）指示的是有多少位被设置为1

6.可用的子网掩码及其相应的CIDR斜线符

/8～/15只可用于A类网络地址。/16～/23只可以用于A类和B类网络地址。而/24～/30则可以用于A、B和C类网络地址。由于A类网络地址可以使用所有的子网掩码并且在网络设计中可以获得最大的灵活性，因此大部分公司都会首选A类网络地址

7.C类地址的子网划分

在一个C类地址中，只有8位是可以用来定义主机的。子网位必须是由左到右进行定义，这中间不能跳过某些位。
| 二进制 | 十进制 | CIDR |
| ——– | —- | —- |
| 10000000 | 128 | /25 |
| 11000000 | 192 | /26 |
| 11100000 | 224 | /27 |
| 11110000 | 240 | /28 |
| 11111000 | 248 | /29 |
| 11111100 | 252 | /30 |
我们不能使用/31或/32，因为我们必须保留至少2位主机位用于主机IP地址的指定

8.C类地址的子网划分，被选用的子网掩码会产生多少个子网

2^x = 子网数目。x是掩码的位数（从第四个八位位组算起）

9.C类地址的子网划分，每个子网有多少个合法的主机号可用

2^y - 2 ＝ 每个子网中的主机数目。y是非掩码位的位数

10.C类地址的子网划分，合法的子网号是什么

256 - 子网掩码 = 块大小，即增量值。从0开始以增量值为分块计数子网掩码数值，就可以得到子网号

11.C类地址的子网划分，每个子网的广播地址是什么

广播地址总是紧邻下个子网的地址，一直是下一个子网前边的地址

12.C类地址的子网划分，在每个子网中，哪些是合法的主机号

合法主机号是那些介于各个子网和广播地址之间的取值，并要减去全0和全1的主机号

13./30掩码的作用

/30掩码永远只提供2个主机地址，不管所拥有的地址是A类、B类或C类，这个掩码几乎就是为点到点链路提供的专用掩码

我们知道B类网络地址有16位可用的主机地址。也就是说我们最多可用14位来进行子网划分（因为必须至少保留两位用于子网中的主机寻址）。使用/16意味着对B类地址并没有进行子网划分，但它仍然是可以使用的掩码

15.B类地址的子网划分，被选用的子网掩码会产生多少个子网

2^x = 子网数目。x是掩码的位数（不同于C类地址的子网划分，从第三个八位位组算起）

16.B类地址的子网划分，每个子网有多少个合法的主机号可用

2^y - 2 ＝ 每个子网中的主机数目。y是非掩码位的位数（不同于C类地址的子网划分，从第三个八位位组算起）

17.B类地址的子网划分，合法的子网号是什么

256 - 子网掩码（非255） = 块大小，即增量值。判断子网掩码（非255）属于第n个八位位组，保留第n-1个八位位组的子网部分，从第n个八位位组起以0开始以增量值为分块计数子网掩码数值，就可以得到子网号（如果n=3，我们在第三个八位位组使用它们，并在第四个八位位组中添加0用于网络地址;如果n=4，我们在第四个八位位组使用它们）

18.B类地址的子网划分，每个子网的广播地址是什么

这个规则和C类地址的子网划分一样，广播地址总是紧邻下个子网的地址，一直是下一个子网前边的地址

19.B类地址的子网划分，在每个子网中，哪些是合法的主机号

这个规则和C类地址的子网划分一样，合法主机号是那些介于各个子网和广播地址之间的取值，并要减去全0和全1的主机号

20.B类地址中使用掩码地址为255.255.255.128（/25）是否使用了C类的子网掩码

这是一个使用了8位来进行子网划分的B类子网掩码，它与C类掩码是完全不同的，实际上它是网络应用中最好的一个子网划分方案，因为它可以创建的子网总数超过500，且每个子网拥有126个主机号的最好组合

21.什么是主机172.16.88.255/20的子网和广播地址

/20代表255.255.240.0，它在第三个八位位组中给出的分块大小是16，由于在第四个八位位组中没有子网位，则在第四个八位位组中可选的答案总是0和255。由于88介于80和96之间，因此子网是80.0，而广播地址是95.255

22.某台路由器在其接口上接收到一个目标地址为172.16.46.191/26的数据包。对这个数据包路由器将做些什么

/26的掩码是255.255.255.192，它给出的分块大小为64，191是介于128和192之间的，并且是128子网的广播地址，对于路由器，在默认时是要丢弃任何一个广播数据包的

23.A类地址子网划分

A类网络的子网划分与B类和C类并没有任何不同，只是在这里有24位可以使用，而在B类网络地址中只有16位，在C类网络地址中只有8位

24.A类地址的子网划分，被选用的子网掩码会产生多少个子网

2^x = 子网数目。x是掩码的位数（不同于B类和C类地址的子网划分，从第二个八位位组算起）

25.A类地址的子网划分，每个子网有多少个合法的主机号可用

2^y - 2 ＝ 每个子网中的主机数目。y是非掩码位的位数（不同于B类和C类地址的子网划分，从第二个八位位组算起）

26.A类地址的子网划分，合法的子网号是什么

256 - 子网掩码（非255） = 块大小，即增量值。判断子网掩码（非255）属于第n个八位位组，保留第n-1个八位位组的子网部分，从第n个八位位组起以0开始以增量值为分块计数子网掩码数值，就可以得到子网号（如果n=2，我们在第二个八位位组使用它们，并在第三和第四个八位位组中添加0用于网络地址；如果n=3，我们在第三个八位位组使用它们，并在第四个八位位组中添加0用于网络地址;如果n=4，我们在第四个八位位组使用它们）

27.A类地址的子网划分，每个子网的广播地址是什么

这个规则和B类、C类地址的子网划分一样，广播地址总是紧邻下个子网的地址，一直是下一个子网前边的地址

28.A类地址的子网划分，在每个子网中，哪些是合法的主机号

这个规则和B类、C类地址的子网划分一样，合法主机号是那些介于各个子网和广播地址之间的取值，并要减去全0和全1的主机号

29.什么是有类路由

路由器设置了某个数值的子网掩码，它会假定在这个有类地址区域内的所有端口都使用相同的子网掩码。

30.有类路由的缺点

由于每台路由器的接口和主机都使用相同的子网掩码，使路由器的接口浪费了一定的地址空间，LAN没有得到足够的地址

31.有哪些路由协议属于有类路由协议

RIPv1和IGRP都被称为有类路由协议，这两个路由协议都没有为子网信息设置字段，因而子网的信息都被丢弃。如果在一个运行RIP和IGRP的网络中混合使用不同长度的子网掩码，这个网络将无法工作

32.无类路由的好处

无类路由选择协议支持子网信息的通告，使用它可以节省大量的IP地址空间

33.有哪些路由协议属于无类路由协议

为了在网络中实现VLSM的设计，就需要所使用的路由协议可以在发送路由更新的同时发送子网掩码信息。这些协议可以是RIPv2、EIGRP和OSPF

34.为什么要给A类网络设计使用VLSM

因为为网络的指定区域创建相邻的地址区域，可以使你很容易实现网络汇总，并使用路由选择协议保持路由更新最小化。

35.什么是汇总路由

汇总，也称为超网，通过将多条需要单独通告的路由信息组合在一个通告中，它可以为路由更新提供更为有效的方式。这样就节省了大量的带宽并缩减了路由器的处理时间。同以往一样，可以使用地址块来配置汇总路由，汇总的主要内容就是块尺寸。

36.IP寻址排错

（1）ping 127.0.0.1。这是一个诊断或环回地址，如果你得到一个成功的ping返回，则可认定你的IP栈是被初始化过的。如果失败，那么你的IP栈失败，并且你需要在这一主机上重新安装TCP/IP
（2）ping本地主机的IP。如果成功，那么可以说明你的网络接口卡（NIC）是功能正常的。如果失败，则表明NIC卡存在问题。这一步并不能说明网线已经连接到NIC上，它只能说明主机上的IP地址栈可以（通过LAN驱动器）与这个NIC进行通信
（3）ping默认网关（路由器）。如果ping正常，表明NIC已经连接到网络并且可以与本地网络进行通信。如果失败，则表明存在一个本地物理网络问题，这个问题可能出现在NIC到路由器之间的任何一个位置上
（4）如果步骤1～3都是成功的，尝试ping一下远端服务器。如果正常，则表明你可以在本地主机与远程服务器之间进行IP通信。同时，你也可以确信远端物理网络也是正常的
如果完成步骤1～4后全都正常，而用户仍不能与服务器通信，那么可能面对某些名字解析的问题，这需要检查你的域名服务器（DNS）的设置。但是，如果ping远端服务器失败，那么你所面对的将是某些远程物理网络的问题，这时，你需要赶到服务器的那边，并执行步骤1～3的测试，直到找出症结所在

37.子网划分的步骤

首先通过使用“256-子网掩码值”来确定你的块尺寸。然后，计数你的子网并确定每个子网的广播地址，它永远是下一个子网之前的数值。合法主机数是子网地址和广播地址之间的数值。默认网关是第一个合法的主机地址

38.理解不同的块尺寸

合法的块尺寸一直都是4、8、16、32、64、128等等，你可以使用“256-子网掩码值”来确定你的块尺寸

5位
